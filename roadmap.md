# Roadmap

## Project Goal
Self-researching CRO board for samsung.com/au. Daily 3-5 CRO ideas with MiniMax mockups. Tinder-like interactive interface.

## Milestones

### M1: Foundation — Research & Architecture
- Scrape and analyze samsung.com/au for page structure, existing components, and CRO opportunities
- Research CRO best practices for e-commerce/electronics
- Define system architecture (tech stack, data model, component tree)
- Design the CRO idea schema (reason, purpose, mockup, category, etc.)

### M2: Research Pipeline — CRO Idea Generation
- Build autonomous research system that analyzes samsung.com/au
- Implement CRO idea generation (LLM-based)
- Store generated ideas in database
- Ensure 3-5 ideas per batch

### M3: MiniMax Image API Integration
- Integrate MiniMax Image API
- Generate mockups for each CRO idea
- Store and serve images

### M4: Frontend — Tinder-like Interface
- Build swipeable card UI for CRO ideas
- Like/Dislike interactions
- Smooth animations and responsive design

### M5: Frontend — Liked & Dislike Pages
- Liked ideas page with grid/list view
- Dislike ideas page
- Move items between liked/disliked (and vice versa)

### M6: Integration & Polish
- End-to-end integration
- Daily batch scheduling
- Error handling, loading states
- Deploy
